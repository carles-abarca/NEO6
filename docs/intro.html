14 de junio de 2025 Introducción v0.2.0 (neo6-proxy + neo6-admin)

Introducción a NEO6

NEO6 es un ecosistema completo de componentes para la interoperabilidad entre aplicaciones modernas y sistemas COBOL legacy (IBM z/OS, Micro Focus, etc.), permitiendo exponer transacciones CICS a través de APIs REST, MQ, TCP y otros protocolos.

Componentes Principales

  • neo6-proxy: Proxy de alto rendimiento escrito en Rust con soporte para múltiples protocolos.
  • neo6-admin: Sistema de administración centralizado con dashboard web y API REST.
  • neo6-protocols: Bibliotecas de protocolos modulares (TN3270, LU6.2, MQ, REST, TCP, JCA).
  • Runtime Environment: Entorno de ejecución completo con despliegue automatizado.

Características Destacadas

  • Arquitectura modular y extensible: Protocolos cargados dinámicamente, configuración flexible.
  • Administración centralizada: Dashboard web, API REST, gestión de múltiples proxies.
  • Monitoreo avanzado: Métricas en tiempo real, logging estructurado, observabilidad.
  • Gestión de configuración: Recarga dinámica, configuración por defecto, overrides por instancia.
  • Control administrativo: Comandos de control, gestión de conexiones, testing de protocolos.
  • Despliegue automatizado: Scripts de deployment, validación automática, estructura de runtime.

Casos de Uso

  • Modernización de aplicaciones mainframe z/OS
  • Integración de sistemas CICS con arquitecturas cloud-native
  • Exposición de transacciones COBOL como APIs REST
  • Conectividad híbrida entre sistemas legacy y modernos
  • Gestión centralizada de múltiples endpoints de integración

Esta documentación cubre la arquitectura completa, los módulos principales, el sistema de administración y la guía de desarrollo para contribuir o integrar NEO6 en tu entorno.
