TCP/IP Proprietary

Protocolo binario personalizado sobre TCP/IP, útil para integraciones de alto rendimiento o requisitos legacy específicos.

Ejemplo de definición en transactions.yaml

transactions:
  TCP01:
    protocol: "tcp"
    server: "tcp-server-1"
    parameters:
      - name: "payload"
        type: "binary"
        required: true
      - name: "destination"
        type: "string"
        required: true
    expected_response:
      status: "ok"
      fields:
        - name: "response_code"
          type: "int"

Ejemplo de implementación

struct TcpHandler;

#[async_trait]
impl ProtocolHandler for TcpHandler {
    async fn invoke_transaction(&self, transaction_id: &str, parameters: Value) -> Result {
        // ...
    }
}

Listener

El listener TCP/IP espera conexiones en el puerto configurado (por defecto 6000). La aplicación cliente debe abrir una conexión TCP y enviar el mensaje según el protocolo propietario definido (puede ser binario o texto estructurado). El listener procesa la petición y responde con el resultado.
